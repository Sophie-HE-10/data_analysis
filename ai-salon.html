ROUND 02 · Hello World
四个词，一条因果链
理解了这条链，后面所有概念都有了根
LLM
Large Language Model
是什么
用互联网级别的文本训练出的大模型，能读懂并生成自然语言
为什么这样
本质是「预测机器」——不是在查资料，是在做概率推断。给定前面所有内容，预测下一个最可能出现的词
处理
文本
→
Token
词元 · 计量单位
是什么
模型处理文本的最小单位，约 1.5 个汉字或 0.75 个英文单词，也是计费的基础
为什么这样
不按字切也不按词切——BPE 算法把高频组合合并，平衡词表大小与语义完整性。计费、速度、上下文限制全以 token 计
堆积
有限
→
Context
上下文窗口
是什么
模型一次对话里能「看到」的全部内容，有 token 数量上限
为什么有限
每次生成都要重算所有历史 token，计算量随长度平方增长——超出窗口的内容，模型直接「忘了」
放进
去的
→
Prompt
提示词 · 你的输入
是什么
你给模型的输入指令
为什么重要
Prompt 就是你往 Context 里放的内容。模型基于这些做概率预测——你放的信息质量，就是输出质量的上限
LLM 用 Token 处理文本 → Token 堆积形成有限的 Context 窗口 → 你的 Prompt 就是你放进窗口的内容 → 写 Prompt 的本质，是在有限空间里最大化有效信息
ROUND 02 · Hello World
写好 Prompt，控制好输出
ANTHROPIC 推荐的 PROMPT 结构
<task>
任务描述
清晰说明你要 AI 做什么，越具体越好
<context>
背景材料
把相关文档、数据、背景塞进来
<instructions>
具体指令
步骤、约束、格式、不能做什么
<examples>
示例
1-3 个输入→输出范例，效果提升最明显
<reminder>
末尾提醒
在结尾再强调最关键的约束，防止模型"忘了"
💡 不是每次都需要五段——task 必须有，examples 效果最显著，其余按需添加。XML 标签帮助 Claude 区分不同内容块，结构越清晰，输出越稳定。
控制输出质量的两个参数
Temperature
随机性 / 创意度
0.0
保守
2.0
随机
低温（0–0.3）：输出稳定可预测，适合数据提取、分类、结构化任务
中温（0.5–0.7）：推荐默认，平衡质量与多样性
高温（1.0+）：更有创意，但容易跑偏，适合头脑风暴
Top-p
核采样 / 候选词范围
模型每步生成时，只从"概率累计到 p"的候选词里随机采样。
Top-p = 0.9 → 去掉低概率长尾，只在最可能的 90% 词里选
Top-p = 1.0：默认值，考虑所有词
Top-p = 0.9：去掉低概率噪声，输出更稳
实践建议：一般只调 Temperature；两个同时调容易相互干扰

ROUND 04 · Agent 家族
从「对话」到「行动」
六个概念，一套完整的 Agent 世界观
Agent
不只是「你问它答」
给它一个目标 → 它自己拆解步骤 → 调用工具 → 完成任务。核心区别：单轮问答 vs 多步自主行动
怎么思考
ReAct
Reasoning + Acting 交替——想一步、做一步、看结果、再想。
适合开放探索型任务，灵活但容易跑偏
Plan & Execute
先把整个任务拆成步骤清单，再按序执行。
适合结构化任务，清晰但计划错了全盘皆输
怎么扩展
Tool
Agent 能调用的外部能力——搜索引擎、代码执行、数据库、第三方 API。
有了 Tool，Agent 才能真正「做事」而不只是「说话」
MCP
Model Context Protocol — Anthropic 推出的开放标准，让任何工具都能被 Agent 统一接入。
类比 USB 接口：统一协议，不用给每个工具单独适配
怎么沉淀
Agent Skill
把调试好的 Agent 能力打包成可复用模块——包含 System Prompt、工具权限、执行逻辑。不用每次重写，一键调用

每步先推理，再做一件事，看到结果再决定下一步——计划随信息实时更新，不预设终态

OPC 设计组 — 一页文档，包含：
· 用户与痛点
· 解决方案
· 市场规模
· 竞争优势

输入

每支团队融资总额 = 机构A + 机构B，单轮上限 200万

模型层

THE MODEL
AI 的"大脑"本身——经过海量数据训练出来的参数集合。这层的词在描述大脑是什么、怎么造出来的、运行时发生了什么。
LLM / 大语言模型 基础模型 微调 Fine-tuning 推理 Inference 幻觉 Hallucination 多模态
💬

对话层

THE CONVERSATION
你和模型"说话"的方式——怎么给指令、怎么控制输出的质量和风格。这层的词在描述怎么跟大脑沟通。
Prompt / 提示词 System Prompt Token Context Window Temperature Few-shot Zero-shot 思维链 CoT
⚙️

工程层

THE SYSTEM
套在模型外面的系统——让 AI 能连接外部信息、完成多步任务、被工程化复用。这层的词在描述给大脑加外挂。
RAG Agent / 智能体 Embedding / 向量 Function Calling MCP Vibe Coding
判断口诀
听到新词，先问自己三个问题：它在说大脑本身（训练、参数、能力）？还是在说怎么跟大脑说话（指令、上下文）？还是在说大脑外挂的系统（检索、工具调用、自动化）？
ROUND 03 · 工具方法
Prompt 的再认识
好的提示词不是一次写成的——是迭代出来的

你写的 Prompt，本质上就是在编程——大模型把「会不会编程」这道门槛移走了，操控 AI 的语言变成了人话。

正确姿势：写 → 看模型在哪里翻车 → 补上那里 → 再写。提示词不是配置项，是你和模型协作的界面。模型裁跟头的地方，恰恰是提示词最该打补丁的地方。

01

结构本身就是信息

一大段文字全堆在一起，模型读着累，理解容易出偏。用标签或括号分段，模型读到每段就知道"这是什么"——不会把背景当任务，也不会把要求当示例。

【背景】我是做产品评测的创作者，读者是开发者。 【任务】为一家公司新发布的产品写评测文章。 【要求】1500字，重点讲实际能不能用，要有判断。 【注意】不要写官方营销话术。
02

顺序错了，前面全白费

不要把所有材料丢给 AI 让它自己想。明确告诉它先做什么、再做什么——顺序本身就是逻辑。让 AI 先看草图后看表单，和先看表单后看草图，结论可能完全不同。

✗ 不好的写法
这是 PRD 和用研报告，帮我看看有什么问题。
✓ 好的写法
第一步：读 PRD，列出所有功能点。第二步：读用研报告，找用户真实诉求。第三步：对比两者，找出设计与需求的矛盾点。
03

2–3 个例子 > 一段规则

规则是抽象的，例子是具体的。模型和人一样，靠具体来理解抽象。与其写一长串"语气要专业、要有数据、不能太长"，不如直接给两三个对照示例——剩下的它会照葫芦画瓢。

✗ 抽象规则
语气要专业，不要太随意，要有数据支撑，不能太长……
✓ 直接给例子
这条评论提取 A 和 B，那条评论提取 C 和 D。剩下的照样做。

痛点 01

记忆太短

每次新对话要重新喂背景；跨天会忘记读过的内容；定时任务会忘记触发。

毕肖峘陈延菲刘玉婷罗健
痛点 02

背景悖论

不喂背景 → 通用废话；喂了背景 → AI 变应声虫，失去批判性，一味顺着你说。

陈延菲许佳宁赵一鑫
痛点 03

从 0 新建容易，改已有基础很难

已有页面小改、已有代码修改、原型迭代，效率都不如直接重新来。

高艺烜单中信黄楚雯
痛点 04

质量不稳定，调试是无底洞

同样的输入不一样的输出；越改越差；调好一个问题冒出新问题。

李知恒黄楚雯王蕗双彩云
痛点 05

只能处理标准场景

规则覆盖到的能做，规则没覆盖的直接束手无策，复杂边缘场景只会转人工。

曹小璐姜怡杨
痛点 06

输出结果难以验证

AI 说的是对的吗？结论有没有依据？在高敏感场景里无法直接使用。

ROUND 05 · Agent & Skill
把好的 Prompt 沉淀成工具
🤖

Agent · 智能体

不只是「你问它答」，而是给它一个目标，它自己拆解步骤、调用工具、完成任务。

类比：助理，你说「帮我安排出差」，它自己查机票订酒店
🔧

Harness · 框架

给 Agent 套上规范，让它在你设定的边界内行动，防止它「乱跑」或做你没要求的事。

类比：工厂 SOP，规定每步怎么做、授权范围是什么、出错了找谁
⚡

Skill · 技能

把反复使用的、经过打磨的 Prompt 包装成一个可以随时调用的「技能卡」，不用每次重写。

类比：公司的 SOP 手册，个人经验变成可复用工具

SKILL 如何对应你们提到的 AI 痛点

每次重新喂背景 → 背景固化在 System Prompt，一次配置永久生效
Prompt 调试成本高 → Skill 是调试好的结果，不用每次从头调
输出格式不稳定 → Skill 里固定输出格式，保证每次结构一致
简单任务描述成本高 → 常用任务做成 Skill，一键触发
